# P2026 火箭航電系統架構說明書

## 1. 模組職責說明 (Module Responsibilities)

### [通訊] InterCoreComm
- **角色**: 核心間通訊中樞 (IPC Hub)。
- **機制**: FreeRTOS RingBuffer。
- **目的**: 將 Core 0 的高頻感測器採集與 Core 1 的邏輯處理實施非同步解耦。

### [感測器] SensorManager
- **角色**: 硬體抽象層 (HAL)。
- **任務**: 
  - `ICM_Task`: 讀取 9 軸慣性數據 (50Hz)。
  - `BMP_Task`: 讀取氣壓高度 (5Hz)。
  - `GPS_Task`: 獲取定位與 GPS 時間同步系統時鐘 (1Hz)。

### [邏輯] DataManager
- **角色**: 數據聚合與分發中心。
- **功能**: 
  - 管理單一 256 位元組緩衝區（存儲與 LoRa 同步傳輸）。
  - 整合 `HealthMonitor` 確保進入狀態機的數據皆經過驗證。

### [健康] HealthMonitor
- **角色**: 故障偵測與隔離 (FDIR)。
- **檢查項目**: 
  - **殘差檢查 (Innovation)**: 偵測感測器實測值與 Kalman 預測值的偏差。
  - **物理檢查 (Physical)**: 偵測數據凍結、超出量程或不合理的變化率。
  - **一致性檢查 (Consistency)**: 跨感測器比對（例如 GPS 與氣壓高度的一致性）。

### [決策] StateMachine
- **角色**: 任務控制指揮官。
- **邏輯**: 管理從點火、動力上升到頂點回收的全流程。
- **安全機制**: 包含 35s 全局頂點超時與 227s 主傘時間保護等防線。

### [存儲/通訊] StorageCommManager
- **角色**: 外部 I/O 管理器。
- **硬體**: SD 卡與 LoRa 無線電。
- **保護**: 使用 Mutex 鎖確保 LoRa 傳輸與接收互斥，防止通訊衝突。

## 2. 模組間封裝與互動機制 (Module Encapsulation & Interaction)

本系統透過嚴格的介面隔離與依賴注入，確保各模組的內部實作細節不會外洩，達成高度解耦：

### [硬體隔離層] SensorManager & StorageCommManager
- **封裝重點**: 隱藏 I2C、UART、SPI 與 GPIO 的暫存器操作。
- **互動機制**: 
  - `SensorManager` 只透過 `InterCoreComm` 拋出標準化的數據結構（如 `ImuData`），不與決策層直接接觸。
  - `StorageCommManager` 內部封裝了 LoRa 通訊的 Mutex 鎖邏輯，外部呼叫者無需擔心底層通訊衝突，只需呼叫 `sendViaRadio`。

### [通訊緩衝層] InterCoreComm
- **封裝重點**: 隱藏 FreeRTOS RingBuffer 的指針操作與核心鎖機制。
- **互動機制**: 提供「生產者-消費者」介面。感測器端僅負責「推入數據」，數據處理端僅負責「取出數據」，兩者互不相識。

### [數據中樞層] DataManager
- **封裝重點**: 封裝了 256B 數據包的累計邏輯、健康檢查的觸發順序以及存儲/傳輸的連動規則。
- **互動機制**: 作為系統的「資訊過濾器」。它接收原始數據，協調 `HealthMonitor` 驗證數據品質後，才將精鍊後的結果餵給 `StateMachine`。

### [診斷隔離層] HealthMonitor
- **封裝重點**: 封裝了卡爾曼殘差、物理極限與跨感測器投票的數學算法。
- **互動機制**: 僅對外暴露簡單的 `isHealthy` 布林狀態，讓呼叫者（DataManager/StateMachine）能專注於「數據能不能用」，而非「為什麼不能用」。

### [決策核心層] StateMachine
- **角色**: 任務控制指揮官。
- **邏輯**: 管理從點火、動力上升到頂點回收的全流程。
- **安全機制**: 包含 35s 全局頂點超時與 227s 主傘時間保護等防線。

## 3. 飛行階段觸發與備援邏輯 (Phase Trigger Logic)

1. **點火 (P5)**: 偵測到垂直加速度 `az > 10.0g` (IMU)。
   - *備援*: 若 IMU 失效，改用氣壓高度上升 `alt > 10m`。
2. **頂點 (P8)**: 偵測到垂直速度翻轉 `vz < -0.2m/s`。
   - *備援*: 點火後 **35 秒強制觸發** 安全超時保護。
   - *限制*: 點火後不滿 10.91 秒禁止進入頂點。
3. **下降段 (P10)**: 頂點觸發後的持續下降確認。
4. **主傘開傘 (P11)**: 高度 `alt < 200m` 且符合 GPS 地理限制。
   - *備援*: 若 GPS 失效，在 200m 時**強制開傘**以保證回收。
5. **任務終止 (P12)**: 落地偵測（高度低、速度低）。
